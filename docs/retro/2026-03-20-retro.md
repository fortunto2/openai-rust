## Failure Patterns

### Pattern 1: Deploy AskUserQuestion Blocking (CRITICAL -- resolved)

- **Occurrences:** 7 wasted deploy iterations (3 in run 1, 4 in run 2)
- **Root cause:** Deploy skill used `AskUserQuestion` to resolve crate name conflicts. In pipeline mode, no user is present -- agent spins retrying the same question each iteration.
- **Wasted:** 7 iterations (70% of all waste)
- **Resolution:** Pipeline prompt now includes explicit "Do NOT use AskUserQuestion" instruction. Runs 3-9 deployed cleanly.
- **Note:** 3rd consecutive project with this defect (supervox, openwok, openai-rust). Marked FIXED in evolution.md.

### Pattern 2: Redo Loops (Build-Review Cycling)

- **Occurrences:** 3 redo cycles (2 in run 2, 1 in run 3)
- **Root cause:** Review found legitimate issues: missing `#[non_exhaustive]`, `serde_json::Value` in public types, typed enum gaps.
- **Wasted:** 0 -- all redo work was productive (real fixes committed each cycle)
- **Assessment:** Expected behavior. Redo limit (2) correctly prevented infinite cycling.

### Pattern 3: Ergonomics Pipeline Interruption (LOW)

- **Occurrences:** 1 (run 7 aborted before first ITER, restarted as run 8)
- **Root cause:** Unknown -- possibly manual interruption or timeout. No ITER line logged.
- **Wasted:** 1 start overhead, no iterations burned
- **Resolution:** Restart worked cleanly -- run 8 completed in 3 iters, 0 waste.
