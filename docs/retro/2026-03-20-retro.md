## Failure Patterns

### Pattern 1: AskUserQuestion Spin-Loop on Deploy

- **Occurrences:** 7 deploy iterations across both runs (iters 3-5 in run 1, iters 2-4 in run 2, iter 9 in run 2)
- **Root cause:** Crate name `openai-rust` was taken on crates.io. Deploy skill used `AskUserQuestion` to ask user for alternative name. In pipeline mode (`--print`), no interactive user to respond. Each iteration re-asked the same question, producing no useful work.
- **Wasted:** 7 iterations (~12 min)
- **Resolution:** Run 1 — user injected message via control file, deploy chose `openai-oxide`. Run 2 — same pattern repeated (3 more wasted iters) until CTRL skip, then resolved on 2nd attempt.
- **Fix:** Deploy skill should auto-detect library projects (check `[lib]` in Cargo.toml) and avoid `AskUserQuestion` in pipeline mode. Should search crates.io for available names and pick the best one automatically. Add `## Library Projects` section to deploy SKILL.md.

### Pattern 2: Review Redo Forced Completion

- **Occurrences:** 2 review iterations, redo limit (2/2) reached
- **Root cause:** Review found README.md version "0.1" → should be "0.2" (minor issue). Output `<solo:redo/>` to trigger fix cycle. After redo limit hit, pipeline forced `<solo:done/>`. The issue (Task 2.1) remains unfixed.
- **Wasted:** 1 iteration (review itself was productive — it found a real issue)
- **Fix:** Review should differentiate MINOR issues (add to plan, don't redo) from CRITICAL issues (redo required). Only output `<solo:redo/>` for build-breaking or security issues.

### Pattern 3: Circuit Breaker Missed Deploy Spin

- **Occurrences:** Should have fired twice (after 3 consecutive deploy failures in each run)
- **Root cause:** Circuit breaker uses hash comparison of iter log output. Each deploy iteration had slightly different output (different AskUserQuestion payloads), so hashes never matched as "identical". The breaker needs a secondary check for consecutive `continuing` results on the same stage.
- **Wasted:** 4 extra iterations (could have stopped after 3 each run)
- **Fix:** Add `same_stage_continuing_count` counter to `solo-dev.sh`. If ≥3 consecutive `continuing` for same stage → trigger circuit breaker.

### Pattern 4: Doubled Signal Output

- **Occurrences:** iter-011-review — `<solo:redo/><solo:redo/>` doubled
- **Root cause:** Agent repeated the signal in summary text after the initial output
- **Impact:** None (cosmetic) — pipeline correctly parsed first signal
- **Fix:** No fix needed.
