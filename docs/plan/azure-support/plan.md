### Decisions Made
- **No generic refactor**: AzureConfig.build() returns a normal `OpenAI` client. Resources untouched. Config trait deferred to separate track.
- **Auth mode enum**: Azure uses `api-key` header, OpenAI uses `Authorization: Bearer`. Need a field to distinguish.
- **api-version as default_query**: Cleanest approach, uses existing infrastructure.
- **Python SDK parity**: Match constructor patterns from `openai-python/src/openai/lib/azure.py`.

### Risks
- Azure `api-key` header vs `Authorization: Bearer` — need to modify `client.rs::request()` which is the hot path for all requests. Keep the branch minimal.
- Dynamic deployment routing (model-based URL rewriting) not included — users must specify `azure_deployment` or use account-level endpoints.
